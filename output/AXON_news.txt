An Indicator represents a queryable entity: a domain, IP address, hash, or raw string. The SDK automatically infers the type from a string, so you can pass raw strings directly to any client method: A ResultSet is the container returned by every query method. It provides: Each query method returns a ResultSet containing typed record objects. Available record types: An EnrichedIndicator is a subclass of Indicator that carries reputation and annotation data directly on the indicator itself. You encounter enriched indicators in two places: 1. Enrichment lookups — client.enrich() returns a ResultSet where each record is an EnrichedIndicator with score, verdict, and annotations: For bulk enrichment, use .keys to iterate the EnrichedIndicator objects: 2. History queries with annotate=True — When you pass annotate=True to dns_history(), extra_history(), host_responses(), or registration_history(), the key and value fields on returned records are upgraded from plain Indicator to EnrichedIndicator when the API provides intel data: Since EnrichedIndicator is a subclass of Indicator, enriched indicators work everywhere a plain indicator does — you can pass them directly to add_to_project(), use them as input to other queries, or access .value and .type as usual. Any method that supports bulk queries can also accept a list of indicators instead of a single value: When you pass a list, the SDK: The returned ResultSet contains one child ResultSet per indicator.
